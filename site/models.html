Danh mục model

Điểm mạnh, điểm yếu và mục đích dùng cho các model Workers AI mà Worker này gọi được. ID có thể thay đổi, xem bản mới tại Cloudflare docs. Đặt allowedModels:"*" để chọn bất kỳ model nào.

Text & chat

@cf/meta/llama-3.1-8b-instructtext default

  • Mạnh: nhanh, rẻ, trợ lý tổng quát ổn định.
  • Yếu: suy luận khó và context dài kém hơn model lớn.
  • Dùng khi: chat mặc định, tóm tắt, phân loại, khối lượng lớn.
curl -s $BASE/v1/text -d '{"prompt":"Tóm tắt 1 câu: ..."}'

@cf/meta/llama-3.3-70b-instruct-fp8-fast

  • Mạnh: suy luận tốt hơn nhiều, tool calling, fp8 nên vẫn nhanh.
  • Yếu: tốn hơn, trễ hơn; context ~24k.
  • Dùng khi: suy luận khó, agent/tool, bản thảo chất lượng cao.

@cf/zai-org/glm-4.7-flash

  • Mạnh: nhanh, đa ngôn ngữ (100+), context 131k, tool calling.
  • Yếu: ít “dày dạn” tiếng Anh như Llama.
  • Dùng khi: chat đa ngôn ngữ (cả tiếng Việt), tài liệu dài, tiết kiệm.

Code

@cf/qwen/qwen2.5-coder-32b-instructcode default

  • Mạnh: chuyên code, sinh/refactor/giải thích đa ngôn ngữ.
  • Yếu: nặng cho snippet nhỏ; kiểm tra ID còn live.
  • Dùng khi: viết hàm, test, refactor, giải thích bug.
curl -s $BASE/v1/code -d '{"prompt":"TypeScript debounce(fn, ms)"}'

@cf/moonshotai/kimi-k2.7-code

  • Mạnh: context 262k, vision + tool + structured output cho agentic.
  • Yếu: đắt nhất, trễ cao.
  • Dùng khi: repo lớn, agent, code kèm ảnh.

Vision & OCR

@cf/mistralai/mistral-small-3.1-24b-instructOCR dày chữ

  • Mạnh: vision + text mạnh, context 128k, trích xuất có cấu trúc tốt; adapter chat-vision.
  • Yếu: lớn/chậm hơn model OCR nhỏ.
  • Dùng khi: screenshot tiếng Việt dày chữ, hóa đơn, biểu mẫu.
curl -s $BASE/v1/extract -d '{"model":"@cf/mistralai/mistral-small-3.1-24b-instruct","imageUrl":"https://.../screen.png","prompt":"OCR toàn bộ"}'

@cf/meta/llama-3.2-11b-vision-instruct

  • Mạnh: suy luận ảnh và captioning tốt.
  • Yếu: cần chấp thuận license Meta 1 lần trước khi dùng.
  • Dùng khi: hỏi đáp ảnh, captioning, suy luận thị giác.

@cf/moondream/moondream3.1-9B-A2Bimage default

  • Mạnh: nhỏ, nhanh, OCR/pointing/detect, rẻ cho khối lượng lớn.
  • Yếu: hallucinate và lặp trên màn hình tiếng Việt dày dấu; hợp ảnh thưa chữ.
  • Dùng khi: caption nhanh, ảnh ít chữ, pointing giá rẻ.

@cf/llava-hf/llava-1.5-7b-hfbeta

  • Mạnh: VQA/captioning nhẹ, mã nguồn mở.
  • Yếu: beta, OCR và tuân lệnh yếu.
  • Dùng khi: thử nghiệm, caption đơn giản.
