Deploy bằng một secret.

Workflow nhận scoped API Token hoặc bộ Account ID + email + Global API Key. Không thêm secret rời.

Chọn credential

Global API Key + email

{"accountId":"...","email":"...","globalApiKey":"..."}

Dùng để triển khai ngay với credential hiện có. Quyền rất rộng, nên chuyển sang token scoped sau khi hệ thống chạy ổn.

Scoped API Token, nên dùng lâu dài

{"accountId":"...","apiToken":"..."}

Cấp Workers Scripts Edit cho đúng account và quyền Workers AI cần thiết.

Lấy thông tin trên Cloudflare

  1. Account ID: vào Account home, mở menu cuối dòng account, chọn Copy account ID.
  2. Email: dùng đúng email của user Cloudflare sở hữu Global API Key, xem tại My Profile.
  3. Global API Key: mở My Profile → API Tokens, phần API Keys, chọn View cạnh Global API Key và xác minh danh tính.
  4. API Token: cùng trang đó, chọn Create Token → Create Custom Token.

Không đưa Global API Key vào code, issue, log, ảnh chụp hoặc tài liệu.

Thêm vào GitHub

  1. Mở Actions secrets.
  2. Chọn New repository secret.
  3. Tên chính xác: CLOUDFLARE_CONFIG_JSON.
  4. Dán JSON trên một dòng, dấu nháy kép thẳng, không dấu phẩy cuối.
  5. Mở Actions, chọn workflow Test, deploy Worker and publish docs, bấm Run workflow.

Sau khi chạy

Bước Deploy Worker in URL workers.dev. Đó mới là API. Test bằng:

curl https://YOUR-WORKER.workers.dev/health

GitHub Pages chỉ phục vụ tài liệu, không xử lý ảnh.
